# file-upload-proj
Проект загрузки файлов на сервер для собеседования на работу.

## Описание
Проект выполнен на Node.js версии 26.2.0.

Ветка итогового результата проекта - `main`.

## Окружение
Файлы `.env` **могут** присутствовать в каждом подпроекте внутри packages: `API` и `Web`. В общем и целом, для каждого параметра `.env` в коде прописаны начальные значения в классах program-data.ts соответствующего подпроекта, поэтому файлы .env не необходимы.

Примеры `.env` (указаны начальные значения, прописанные в коде):
* `API`:
    ``` bash
    PORT="4001" # Отвечает за запуск сервера на указанном порте
    UPL_DIR="uploads" # Отвечает за сохранение бинарников в указанной директории
    META_DIR="data" # Отвечает за сохранение метаданных в указанную директорию
    META_FILE="files.json" # Отвечает за сохранение метаданных в указанный файл
    ```
* `Web`:
    ``` bash
    NEXT_PUBLIC_API_URL="http://localhost:4001" # Отвечает за обращение клиента к указанному API
    ```

## Запуск
1. Для установки необходимых зависимостей проекта выполняется команда:
    ``` bash
    npm install
    ```
2. Для запуска серверной части:
    ``` bash
    npm run server # Выполняет скрипт внутри подпроекта API, который в свою очередь запускает nodemon (запуск сервера)
    ```
3. Для запуска клиентской части:
    ``` bash
    npm run client # Выполняет скрипт внутри подпроекта Web, который в свою очередь запускает next
    ```
Также можно добавить после команд npm run символ `&`, чтобы запустить процессы в фоновом режиме (по крайней мере на Linux это работает так, не помню как на Windows). Однако мне было удобнее запускать их в терминалах в режиме split

Также на скорость выполнения повлияла подготовка к сессии и проблемы с впн.

При наличии дополнительного времени - потратил бы его на внедрение Rate limit, на оформление css компонентов React и фильтр по MIME.
